**本项目所有图片均由AI生图模型自动生成**，不依赖人工收集。利用当前高精度生图模型（如Seedream、Midjourney等），通过精心设计的Prompt模板，批量生成符合中医药专业标准的图片资源。

### 5.1 两层图片架构（纯中医游戏，无需显微镜视角）
- [x] 设计两层图片数据模型
- [x] 生成原植物图片（18味药 × 1张）- **3D渲染风格，透明背景** ✅
- [x] 生成饮片图片（18味药 × 1张）- **3D渲染风格，透明背景** ✅

### 5.2 生图Prompt设计（3D渲染风格 + 透明背景）
每种药物类型需要专门的生图Prompt：

**原植物图Prompt模板：**
```
3D rendering style {latinName} plant, {morphology}, {keyFeatures},
transparent background, clean cutout, studio lighting, high detail,
no text, no watermark, game asset, PNG style, 8K resolution
```

**饮片图Prompt模板：**
```
3D rendering style {latinName} dried herb, {specification}, {appearance},
{crossSection}, transparent background, clean cutout, studio lighting,
high detail, no text, no watermark, game asset, PNG style, 8K resolution
```

**五行场景背景Prompt模板：**
```
东方幻想风格游戏场景背景，[五行区域名称]，
[季节特征]，[五行色彩]，
[环境元素描述]，[氛围描述]，
新国风水墨渲染，中国传统山水画意境，
适合游戏UI背景，层次感强，景深效果，
超高清，8K，游戏美术资源
```

## 备注

### 核心策略

#### 1. AI生图全自动化策略
- **不人工收集图片**：所有药物图片、场景背景均由AI生图模型自动生成
- **高精度Prompt工程**：由专业美术设计精心设计的生图Prompt模板
- **批量自动化**：开发批量生图脚本，一键生成50味药的三层图片
- **AI质量验证**：使用Qwen-VL自动验证生图质量，筛选可用图片
- **成本优化**：相比传统游戏美术外包，AI生图成本降低90%以上

#### 2. AI自动化测试策略
- **无传统单元测试**：不编写传统测试用例，完全由AI多模态模型驱动测试
- **端到端验证**：模拟真实用户操作路径，验证功能和视觉效果
- **视觉回归测试**：AI自动检测UI变化，标记视觉差异
- **持续集成**：每次代码提交自动触发AI测试，生成测试报告
- **智能问题定位**：AI自动分析测试失败原因，给出修复建议

### 技术债务
1. 需要统一错误处理机制
2. 需要添加单元测试（已由AI自动化测试替代）
3. 需要优化图片加载性能

### 设计决策
- 使用AI生成真实药图替代人工收集，更专业且可控
- 性味归经玩法替代问答玩法，策略性更强
- 方剂追缉令提供明确目标导向
- AI自动化测试替代传统测试，覆盖率更高
