## 一、设计概述

### 1.1 设计目标
为《方灵旅记》第一阶段（里程碑1）设计完整的用户交互系统，包含从游戏启动到药灵收集、图鉴查看的全流程交互细节。

### 1.2 设计理念
- **东方幻想美学**：水墨意境 + 水晶元素 + 微光粒子
- **沉浸式探索**：无缝场景过渡，自然的交互反馈
- **低门槛高留存**：操作简单，视觉反馈丰富
- **渐进式引导**：新手引导不突兀，融入场景

### 1.3 核心交互循环
```
启动游戏 → 主菜单 → 进入山谷 → 探索/收集 → 查看图鉴 → 继续探索
     ↑_________________________________________________|
```

### 2.1 游戏启动画面

#### 2.1.1 界面描述
**画面构成**：
- **背景层**：动态水墨山水画，缓慢流动的云雾效果
- **中层**：游戏Logo "方灵旅记" 以书法字体呈现，金色描边

```

│                                                             │
│                       点击开始                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 2.1.2 交互行为
| 状态 | 表现 | 触发条件 |
|------|------|----------|
| **初始态** | Logo淡入，云雾缓慢流动，粒子漂浮 | 画面加载完成 |
| **悬停态** | "点击开始"文字微光闪烁 | 鼠标移入画面 |
| **点击态** | Logo发光扩散，云雾加速流动 | 点击画面任意位置 |
| **过渡态** | 画面淡出至黑色，持续800ms | 点击确认 |

#### 2.1.3 动画细节
- **Logo动画**：从透明度0到1，持续1.5s，缓动函数 ease-out
- **云雾流动**：CSS animation，translateX循环，每轮20s
- **粒子漂浮**：随机位置生成，上下浮动 ±15px，周期3-5s随机
- **点击反馈**：Logo缩放1.0→1.1→1.0，时长300ms

**按钮样式表**：

#### 2.2.3 菜单选项功能

**① 开始旅程**
- 功能：创建新存档，进入新手引导
- 点击后：弹出存档确认对话框（覆盖当前存档警告）
- 二次确认后：播放开场动画，进入药灵山谷

**② 继续修行**
- 功能：加载最近的存档
- 禁用条件：无存档数据时显示为灰色
- 加载过程：显示加载动画（旋转的药灵种子）

**③ 设置**
- 功能：打开设置面板（覆盖层）
- 设置项：音量、画质、语言、账号

**④ 退出**
- 功能：退出游戏
- 点击后：弹出确认对话框
- 确认后：淡出并关闭窗口

#### 2.2.4 动画序列

**菜单入场动画**：
```
1. 背景模糊药灵山谷淡入 (0-500ms)
2. Logo从上滑入+淡入 (200-800ms)
3. 按钮依次从下方滑入，间隔100ms (500-1200ms)
4. 装饰元素水墨晕染展开 (800-1500ms)
```

**按钮选中动画**：
```
1. 当前按钮边框发光扩散
2. 其他按钮透明度降至50%
3. 点击后按钮"按下"效果（translateY +2px）
4. 场景过渡效果
```

## 四、药灵种子交互设计

### 4.1 种子视觉设计

#### 4.1.1 种子形态
```
【种子结构】

     ╭─────────────╮
    ╱   ★★★★★    ╲     ← 顶部光晕
   │  ╭─────────╮  │
   │  │ ╭─────╮ │  │
   │  │ │ ◉◉◉ │ │  │    ← 水晶内核（半透明）
   │  │ │ ◉◉◉ │ │  │       内部可见药材轮廓
   │  │ ╰─────╯ │  │
   │  ╰─────────╯  │
    ╲_____________╱
         │   │
         └─┬─┘         ← 底部微光粒子
           ↓
      (轻微上下浮动)
```

**未收集状态**：
- 外观：半透明水晶球，内部模糊轮廓
- 光效：微光呼吸效果（亮度周期变化）
- 标签：显示"???"或草药轮廓图标

**已收集状态**：
- 外观：金色光芒包裹，内部清晰可见
- 光效：持续稳定发光
- 标签：显示药名（如"麻黄"）

#### 4.1.2 种子状态变化

| 状态 | 视觉效果 | 交互性 |
|------|----------|--------|
| **未被发现** | 完全透明，不可见 | 无 |
| **可收集** | 半透明闪烁 | 可点击 |
| **悬停** | 发光增强+上浮 | 点击收集 |
| **收集动画** | 破碎→光芒汇聚 | 无 |
| **已收集** | 金色稳定发光 | 点击查看详情 |

2. 视差层以不同速度移动

## 十一、技术实现建议

### 11.1 动画技术选型

| 动画类型 | 推荐技术 | 理由 |
|----------|----------|------|
| UI过渡 | Framer Motion | React生态，声明式API |
| 粒子效果 | Canvas 2D | 性能友好，可控性强 |
| 视差滚动 | CSS transform + IntersectionObserver | 流畅，硬件加速 |
| 复杂序列 | GSAP | 时间线控制精确 |
| 微交互 | CSS animations | 简单高效 |

### 11.2 性能优化建议

```
【渲染优化】
- 使用 transform 和 opacity 进行动画
- 对动画元素启用 will-change
- 视差层使用 CSS containment
- 粒子系统使用对象池

【资源优化】
- 图片懒加载
- 种子图标使用 SVG
- 背景图分区域加载
- 动画资源预加载

【交互优化】
- 点击响应 < 100ms
- 动画帧率 > 55fps
- 支持 reduce-motion 媒体查询
```

### 11.3 响应式适配

```
【断点设计】
- Desktop: ≥1280px (完整体验)
- Tablet: 768px-1279px (简化视差)
- Mobile: <768px (单列布局)

【移动端适配】
- 种子点击区域增大至 60px
- 滑动切换区域
- 简化粒子效果
- 底部导航栏
```
