## 一、核心发现：架构理解错误

### 1.1 错误的理解（之前的假设）

之前的假设是：
- ValleyScene 和 GatheringStage 是两套并存的系统
- 需要从 ChapterEntry 添加导航到 GatheringStage
- 问题是导航入口缺失

**这是完全错误的！**

### 1.2 正确的 v3.0 架构

根据 `03-chapter-system.md` 和 `药灵山谷v3.0_融合设计.md`：

**章节化学习是 v3.0 的核心**，每章有**6个阶段**的线性流程：

```
章节选择 (ChapterSelect)
    ↓ 点击进入某章
章节入口 (ChapterEntry) - 显示章节信息，"开始本章"按钮
    ↓ 点击"开始本章"
阶段1: 师导入门 (MentorIntroStage) - AI导师导入
    ↓ 完成
阶段2: 山谷采药 (GatheringStage) - 地图探索+采集小游戏 ⭐ Phase 2
    ↓ 完成
阶段3: 药灵守护 (BattleStage) - 打字战斗 ⭐ Phase 3
    ↓ 完成
阶段4: 方剂学习 (FormulaLearningStage) - AI讲解
    ↓ 完成
阶段5: 临床考核 (ClinicalStage) - 病案辨证
    ↓ 完成
阶段6: 开放世界解锁 (OpenWorldStage)
    ↓ 完成
返回章节选择，解锁下一章
```

**GatheringStage 不是独立页面，是章节流程的第2阶段！**

## 二、问题根因分析

### 2.1 ChapterEntry 完全错误

**当前实现 (ChapterEntry.tsx)**：
- 显示标签页导航（"山谷场景"/"本章药材"/"本章方剂"）
- "山谷场景"标签渲染 ValleyScene（v2.0 遗留的种子系统）
- "开始学习"按钮无事件绑定
- 没有"开始本章"按钮

**设计文档要求**：
- 应该显示章节概览（名称、五行、4味药材、2个方剂）
- 一个显眼的"开始本章"按钮
- 点击后进入阶段1（师导入门），不是 ValleyScene

**ValleyScene 的问题**：
- ValleyScene 是 v2.0 的"水晶球种子系统"
- 在 v3.0 中应该被**完全移除**，不是修改
- 文档明确说："v2.0的点击水晶球玩法被替换为山谷采药冒险"

### 2.2 GatheringStage 孤立

**当前状态**：
- GatheringStage 已实现（等角地图+小游戏）
- 有独立路由 `/chapter/:chapterId/gathering`
- 但没有任何入口可以进入
- 没有与章节流程连接

**正确位置**：
- GatheringStage 应该是阶段2的组件
- 由 StageManager 渲染和管理
- 不是通过 URL 直接访问

## 三、修复方案（重大重构）

### 3.1 方案概述

这不是简单的修复，是**架构重构**：
1. 移除 v2.0 遗留的 ValleyScene
2. 重构 ChapterEntry 为章节入口
3. 创建 StageManager 管理6阶段流程
4. 将 GatheringStage 整合为阶段2

## 七、结论

### 关键洞察

1. **Phase 2 的实现是正确的**，但放在错误的位置
   - GatheringStage 应该作为章节流程的阶段2
   - 而不是独立的可访问页面

2. **ChapterEntry 完全错误**
   - 应该显示章节概览 + "开始本章"按钮
   - 而不是 ValleyScene（v2.0遗留）

3. **缺少 StageManager**
   - 这是 v3.0 章节化学习的核心
   - 管理6阶段流程和状态流转

4. **测试策略错误**
   - 不应该直接访问 GatheringStage URL
   - 应该测试完整的章节流程
