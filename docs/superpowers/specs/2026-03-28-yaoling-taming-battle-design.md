# 药灵驯服战 - 详细设计文档

## 1. 游戏概述

**游戏名称**：药灵驯服战
**核心玩法**：玩家通过回答中医药知识问题，帮助失忆的药灵回忆自己的属性，从而驯服药灵。

### 1.1 背景设定
药灵们在山谷中修炼太久，渐渐忘记了自己的属性（性味、归经、功效等）。玩家需要回答药灵们的问题，帮助它们恢复记忆。

### 1.2 游戏特色
- **人物化药灵**：每个药灵都是拟人化的角色，有独特形象和性格
- **开放式问答**：支持自由回答，AI评分（1-5分）
- **失忆主题**：药灵问的是自己忘记的属性，而非让玩家猜测身份
- **视觉反馈**：驯服进度条、星光飞散特效、AI生图药灵形象

- **激活药灵**：蓝色光环、1.1倍放大、连接线指向

## 3. 核心机制

### 3.1 单问题激活机制
- 同一时间只有一个药灵处于提问状态
- 点击药灵切换激活目标
- 切换时问题泡泡平滑移动

### 3.2 四种问题类型

| 类型 | 描述 | 示例 | 分值 |
|------|------|------|------|
| **回忆型** | 药灵忘记自己的属性 | "我归哪几条经？" | 1-5分 |
| **判断型** | 确认某种说法 | "我味道很苦，对吗？" | 1分 |
| **选择型** | 从选项中选择 | "我的功效是？A.补血 B.解表" | 2分 |
| **自由型** | 开放式深度问题 | "我和川芎有什么不同？" | 1-5分 |

### 3.3 AI评分系统（1-5分）
- **5分**：完美回答，核心知识+扩展
- **4分**：正确，包含核心知识
- **3分**：基本正确，缺少细节
- **2分**：部分正确
- **1分**：错误

分数转换为驯服进度：5分→25%，4分→20%，以此类推。

### 3.4 技能系统

| 技能 | 效果 | 冷却 |
|------|------|------|
| 🌟 灵光一闪 | 显示部分提示（首字/字数） | 30s |
| 📖 典籍查阅 | 显示药材详细描述 | 45s |
| 💡 青木提示 | 青木先生直接给出答案，得分减半 | 60s |

## 4. 药灵形象设计

### 4.1 设计理念
- **人物属性**：拟人化角色，非草药本身
- **药物特征融入**：服装、配饰、颜色体现药物特点
- **三种等级**：普通（可爱）、精英（优雅）、王者（威严）

### 4.3 代表性药灵设计

#### 当归灵（补血圣药）
- **形象**：红衣少女，手持红色丝带
- **特征**：红色主调，温婉笑容，裙摆如花瓣
- **性格**：温和、乐于助人
- **台词**："我总是记不住事情...你能告诉我吗？"

#### 川芎灵（血中气药）
- **形象**：青衣少年，手持折扇
- **特征**：青色主调，活泼眼神，飘逸发带
- **性格**：活泼、有点小骄傲
- **台词**："喂！你知道我能治什么病吗？"

#### 黄芪灵（补气之长）
- **形象**：黄袍老者，长须飘飘
- **特征**：金色主调，威严面容，手持药杖
- **性格**：威严、慈爱
- **台词**："小友，老夫忘了自己的功效，你可知晓？"

## 5. 数据结构

```typescript
interface MedicineSpirit {
  id: string;
  medicineId: string;           // 关联药材ID
  name: string;                 // 药灵名称（如"当归灵"）
  displayName: string;          // 显示名称（如"当归"）
  imageUrl: string;             // AI生成的形象URL
  difficulty: 'normal' | 'elite' | 'boss';

  // 状态
  position: { x: number; y: number };
  tameProgress: number;         // 0-100 驯服进度
  state: 'floating' | 'asking' | 'tamed' | 'escaped';

  // 当前问题
  question: SpiritQuestion;
}

interface SpiritQuestion {
  id: string;
  type: 'recall' | 'judge' | 'choice' | 'free';
  question: string;             // 问题文本
  options?: string[];           // 选择型问题的选项
  acceptableAnswers: string[];  // 可接受的答案
  hint: string;                 // 提示
}

interface AnswerEvaluation {
  score: 1 | 2 | 3 | 4 | 5;
  isCorrect: boolean;           // >=3分算正确
  feedback: string;             // 反馈文本
  bonusInfo?: string;           // 额外知识
}

interface BattleState {
  wave: number;
  totalWaves: number;
  spirits: MedicineSpirit[];
  activeSpiritId: string | null;
  score: number;
  combo: number;
  timeElapsed: number;
  tamedCount: number;
}
```

## 6. 动画与特效

### 6.1 药灵漂浮动画
- 上下浮动：y轴 [-15, 0] 循环，3秒周期
- 呼吸发光：boxShadow脉冲效果
- 相位差：每个药灵不同的延迟

### 6.2 驯服特效
1. 药灵发光（金色）
2. 弹出✓标记
3. 化为星光粒子
4. 飞向右上方"药囊"
5. 药囊数字+1

### 6.3 问题泡泡动画
- 出现：从药灵位置弹入
- 切换：平滑移动到新药灵
- 反馈：正确变绿/错误变红

## 7. 游戏流程

```
初始化
  ↓
生成药灵（调用LLM生成问题）
  ↓
玩家点击药灵激活
  ↓
输入答案
  ↓
LLM判断并评分
  ↓
更新驯服进度
  ↓
进度满100% → 驯服动画
  ↓
所有药灵驯服 → 下一波次
```

## 8. 技术要点

### 8.1 LLM调用策略
- **预生成**：波次开始时一次性生成所有问题
- **缓存**：已驯服药灵的问题-答案对本地缓存
- **降级**：LLM不可用时使用本地题库

### 8.2 图片生成策略
- **预生成**：常用药材的药灵形象预生成
- **按需生成**：罕见药材按需调用Seedream
- **缓存**：生成的图片URL本地存储

### 8.3 性能优化
- 药灵图片懒加载
- LLM响应缓存
- 动画使用CSS transform而非position
