## 评价标准

### 总分100分

| 维度 | 权重 | 满分 | 说明 |
|------|------|------|------|
| A1. 上下文连贯性 | 10% | 10分 | 多轮对话保持上下文 |
| A2. 教学逻辑性 | 10% | 10分 | 逻辑严密，层层递进 |
| A3. 角色一致性 | 10% | 10分 | 保持"青木先生"角色 |
| A4. 表达生动性 | 10% | 10分 | 故事化，引人入胜 |
| B1. 苏格拉底引导 | 15% | 15分 | 引导而非直接给答案 |
| B2. 知识点准确性 | 10% | 10分 | 中医理论准确 |
| B3. 学习适应性 | 10% | 10分 | 适配学生水平 |
| C. 用户体验 | 10% | 10分 | 响应速度等 |
| D. 题目质量 | 15% | 15分 | 多样性、解析质量 |

### 等级标准

| 等级 | 分数 | 说明 |
|------|------|------|
| S级 | ≥90 | 优秀，可直接上线 |
| A级 | 80-89 | 良好，小优化后可上线 |
| B级 | 70-79 | 及格，需改进 |
| C级 | 60-69 | 不及格，需大幅改进 |
| D级 | <60 | 严重问题，需重做 |

**通过标准：≥B级（70分）**

## 测试场景

### 场景1：章节入门对话
- 测试欢迎语生成
- 测试角色一致性
- 测试学习目标说明

### 场景2：苏格拉底引导（核心）
- Round 1: 学生答错 → 期待引导思考
- Round 2: 学生仍困惑 → 期待更多提示
- Round 3: 学生要求答案 → 期待答案+讲解

### 场景3：多轮对话连贯性
- 进行5轮连续对话
- 验证上下文引用
- 验证话题深入

### 场景4：题目生成
- 测试题目多样性
- 测试选项设计
- 测试解析质量

### 场景5：不同学生类型
- 学渣型：经常答错，需要多次引导
- 普通型：有时对有时错
- 学霸型：大多能答对，喜欢深入

## 反馈优化循环

```
测试 → 评分 → 问题分类 → Prompt优化 → 重新测试
 ↑                                              |
 └──────────────────────────────────────────────┘
```

### 终止条件
- 连续2轮达到S级 → 终止
- 达到5轮未达B级 → 终止并报告问题
- 累计10轮 → 终止

### 优化方向

**P0级（严重）问题**：
- 知识点错误 → 修正System Prompt，添加知识库
- 完全脱离角色 → 强化角色定义
- 逻辑混乱 → 添加Few-shot示例

**P1级（重要）问题**：
- 引导策略不当 → 优化引导Prompt
- 题目重复率高 → 添加多样性约束
- 上下文断裂 → 优化上下文管理

**P2级（优化）问题**：
- 表达不够生动 → 添加修辞要求
- 响应速度慢 → 优化缓存策略

## 注意事项

1. **API配额**：AI评估模式消耗API配额，请合理规划
2. **响应时间**：AI评估可能需要较长时间（每轮5-10秒）
3. **离线优先**：开发阶段使用离线模式，发布前使用AI评估
4. **结果波动**：AI评估结果可能有±5分波动，建议多次测试取平均
